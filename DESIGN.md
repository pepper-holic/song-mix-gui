# DESIGN.md — song-mix-gui

이 문서는 현재 구현된 UI/UX를 있는 그대로 기록한 **디자인 시스템 현황판**이다.
새 기능을 짤 때 "이미 있는 패턴"을 재사용하고, 개선 작업을 시작하기 전 기준선으로 삼기 위한 문서.
추측이나 희망사항은 배제하고 `app/frontend/src/`의 실제 코드(styles.css, App.tsx, SongPane.tsx,
ChannelNode.tsx 등)에서 역산했다.

## 1. 제품 성격과 원칙

- **위상**: Studio One을 대체하지 않는 **보조 데스크톱 도구**. 믹스 구조를 읽고, 검증하고,
  다른 곡에 옮기는 것이 전부 — 오디오 재생/편집 기능 없음. 사용자는 엔지니어 1인(제작자 본인),
  세션당 여러 곡을 동시에 열어놓고 비교/전송하는 워크플로우가 핵심.
- **정보 밀도 우선**: 마케팅적 여백보다 그래프 위에서 채널·인서트·라우팅을 한눈에 읽는 것이 우선.
  진성 데스크톱 앱 문법(툴바+탭+스테이터스바)을 따르지 브라우저스러운 카드 레이아웃을 쓰지 않는다.
- **되돌릴 수 있어야 한다**: 모든 쓰기 작업(전송/일괄적용/체인교체)은 Undo 가능해야 하고, 위험한
  작업(덮어쓰기, 체인 교체, 중첩 배제 경고)은 확인 모달을 거친다 — CLAUDE.md의 "원본 절대 수정
  금지" 철칙이 UI 레벨에서도 "먼저 물어보고, 되돌릴 수 있게" 형태로 이어진다.
- **다크 테마 전용**: 라이트 모드 없음(의도적 — 오디오 프로덕션 도구는 다크 테마가 업계 표준이고
  장시간 사용 시 눈 피로 감소가 실사용 요구). 테마 토글 요구 없었음.
- **자동 유사매칭 금지 원칙의 UI 반영**: 일괄 적용처럼 자동 판단이 위험한 곳(라벨 매칭)은 항상
  사용자에게 명시적 매칭표를 보여주고 확인받는다 — "똑똑하게 추측"보다 "정확하게 보여주고 선택".

## 2. 화면 구조 (Information Architecture)

```
┌─────────────────────────────────────────────────────────────┐
│ toolbar: [파일 그룹] | [보기·편집 그룹] | [일괄적용 그룹] | 범례       │
├─────────────────────────────────────────────────────────────┤
│  <PaneGroup> — VSCode식 재귀 분할 트리(app/frontend/src/paneLayout.ts) │
│  leaf(탭바+그래프) 하나 또는 leaf/split이 임의 깊이로 중첩         │
│   - leaf: tabs(그 leaf가 보유한 문서만) + SongPane(활성 탭)      │
│     - pane-title, pane-search(검색+필터), React Flow 그래프     │
│     - DetailPanel(선택 시 우측 슬라이드인 오버레이)              │
│   - split(row|column): 리사이즈 핸들로 구분된 자식 2개 이상       │
├─────────────────────────────────────────────────────────────┤
│ statusbar: [스피너] 상태 메시지          [프리웜 진행률]         │
└─────────────────────────────────────────────────────────────┘
```

- **탭 = 열린 문서, 탭바는 leaf(패널)마다 독립** — 2026-07-15부터 앱 전역 한 줄이 아니라
  `PaneGroup.tsx`가 재귀 렌더링하는 각 leaf가 자신만의 탭바를 가짐(VSCode의 에디터 그룹과 동일
  모델). 문서 하나는 항상 정확히 하나의 leaf에만 속한다.
- **자유 분할**: 탭을 다른 패널 가장자리(상/하/좌/우 25%)로 드래그하면 그 방향으로 분할 생성,
  중앙으로 드래그하면 그 leaf로 이동. 우클릭 컨텍스트 메뉴("왼쪽/오른쪽/위/아래로 분할", "탭
  닫기")가 드래그의 안정적 대안이자 유일한 자동 회귀 검증 경로(`app/e2e_test.py`) — 좌표 계산
  없이 항상 동작. split의 자식 사이 리사이즈 핸들로 비율 조정 가능. 이전의 고정 좌/우 2패널
  토글("좌우 스플릿" 버튼)은 완전히 대체되어 제거됨.
- **패널(leaf) 내부에 그래프+검색+상세패널이 전부 중첩** — DetailPanel은 leaf를 덮는 오버레이라
  그래프 레이아웃을 밀어내지 않는다(모달리스, 그래프 보면서 파라미터 확인 가능).
- **모달은 전부 화면 중앙 오버레이**(`modal-backdrop` + `modal`), 배경 클릭/Esc로 닫힘 — 충돌,
  체인교체 확인, 트랙전송 확인, 단축키 도움말, 비교결과, 일괄적용 다이얼로그 전부 동일 패턴 재사용.

## 3. 디자인 토큰 (`styles.css` `:root`)

2026-07-15 후반 세션에서 "너무 둥글둥글한 SaaS풍" 인상을 지우고 Studio One 콘솔에 가까운
뉴트럴 차콜 그레이 + 앰버 액센트로 재설계(각진 모서리, 블러/글로우 제거). 이전 팔레트(코발트
블루 배경 + 시안 액센트, 라운드 4/7/10px, 모달 backdrop-blur)는 git 히스토리에 남아있음.

`output` #3c4d3e(그린그레이) — 새 뉴트럴 팔레트와 어울리도록 채도를 낮춤).
`.modal-backdrop`의 `backdrop-filter: blur()`도 제거(플랫 톤 유지, 불투명도만으로 배경 차단).

## 4. 컴포넌트 패턴

- **버튼**: 툴바/모달/BulkApply 헤더가 전부 같은 규칙 공유 — surface-2 배경, hover 시
  surface-3+accent 테두리, active 시 `scale(0.97)`, disabled는 opacity 0.45. 아이콘은
  `Icons.tsx`의 stroke SVG(currentColor 상속이라 버튼 색과 자동 동기화).
- **모달**: `modal-backdrop`(blur+fade) → `modal`(scale+slide fade-in). 파괴적 확정 버튼은
  항상 액션 그룹의 **첫 번째** 자식이고 danger 색상(`.modal-actions button:first-child`) —
  즉 "위험한 버튼이 왼쪽/첫 번째"라는 암묵적 컨벤션이 이미 코드에 있음(신규 모달 작성 시 순서 주의).
- **채널 노드(그래프)**: kind별 배경색 + insert 배지(최대 6개, 순번 표시) + draggable 힌트
  아이콘(⠿). React Flow의 `nodrag nopan` 클래스로 네이티브 HTML5 드래그와 React Flow 패닝을
  공존시킴(이 프로젝트 고유의 까다로운 트릭 — 재발견 비용 있음).
- **디테일 패널**: 우측 320px 오버레이, 인서트 리스트+파라미터 테이블(정렬 가능 컬럼), 해석불가
  플러그인은 warning 배지.
- **빈 상태(empty state)**: 아이콘+제목+CTA 버튼 패턴(`empty-pane`), 두 패널 모두 통일.
- **상태바 색상 파생**: 별도 상태값 없이 `statusMsg` 문자열에 "실패/오류"/"완료/복원됨/복사됨/열림"
  키워드가 있는지로 색을 유추(`App.tsx` statusKind). 값싸지만 취약한 패턴 — 새 상태 메시지 문구를
  추가할 때 이 키워드 목록도 같이 갱신해야 함(누락되면 색이 안 붙을 뿐 크래시는 안 남).

## 5. 인터랙션 패턴

| 동작 | 트리거 |
|---|---|
| 복사 | 노드 우클릭 (버스/FX/트랙만) 또는 선택 후 `Ctrl+C` |
| 붙여넣기 | 패널 빈 곳 우클릭 또는 `Ctrl+V` |
| 전송 | 노드를 다른 패널로 드래그 앤 드롭 |
| 체인 복사(교체용) | `Ctrl+우클릭` → 대상 채널 우클릭 시 교체 확인 모달 |
| 비교 기준 지정 / 비교 실행 | `Shift+우클릭` (A 지정) / `Alt+우클릭` (A와 비교) |
| 실행 취소 | `Ctrl+Z` 또는 툴바 버튼(포커스된 패널 대상) |
| 검색/필터 | 패널 상단 입력창, 필터 체크 시 비매칭 노드 dim |
| 단축키 도움말 | `?` 키 (입력창 포커스 중엔 무시) |
| 모달/드롭다운 닫기 | `Esc` 전역, 또는 배경 클릭 |
| 패널 분할 | 탭을 다른 패널 가장자리로 드래그, 또는 탭 우클릭 → 방향 선택 |
| 패널 이동/병합 | 탭을 다른 패널 중앙으로 드래그 |
| 패널 크기 조정 | split 자식 사이 리사이즈 핸들 드래그 |
| 탭 닫기 | 탭의 × 클릭 (마지막 탭이면 그 패널은 자동으로 사라지고 트리가 평탄화) |

패널 포커스(`focusedLeafId`)가 Undo/BulkApplyDialog 기본 소스 등 "어느 문서에 적용할지" 애매한
액션의 타겟을 결정 — 노드 클릭 시 그 leaf에 포커스를 강제로 옮겨서 그 직후 `Ctrl+C`가 바로
먹게 하는 트릭이 있음. 자유 분할 도입 이후에는 좌/우 같은 고정 역할이 없으므로 "가장 최근에
상호작용한 패널"이 곧 포커스 타겟이라는 점이 이전보다 더 부각됨(예: Undo 대상은 항상 최근에
클릭/포커스한 패널의 문서).

## 6. 접근성 현황 (있는 그대로)

- `:focus-visible`로 키보드 포커스만 표시(마우스 클릭 시 안 보임) — 구현됨.
- 아이콘 전용 버튼도 대부분 텍스트 라벨을 동반(`<span className="icon">…</span>텍스트`) —
  스크린리더에 노출되는 접근 가능한 이름 있음.
- **갭**: React Flow 그래프 자체는 캔버스 기반이라 키보드만으로 노드를 순회/선택/드래그할 방법이
  없음(마우스 전용 워크플로우). `role`/`aria-label`이 커스텀 인터랙티브 요소(모달, 드롭다운)에
  명시적으로 붙어있지 않음 — 시맨틱 HTML(`<button>`, `<input>`)에 의존해 최소한은 확보되지만
  포커스 트랩(모달 내부에 포커스 가둠)은 없음.
- **갭**: 라이트 테마/고대비 모드 없음 — 단일 사용자 개인 도구라는 현재 범위에선 우선순위 낮음.

## 7. 알려진 UX 갭 / 개선 후보

### 2026-07-15 개선 패스에서 해결됨

WCAG 2.2 / WAI-ARIA APG / NN/g 휴리스틱 / Material Design 가이드를 참고 기준 삼아 아래 7개 중
6개를 처리했다(상세: `graphNav.ts`, `SongPane.tsx`, `ChannelNode.tsx`, `BulkApplyDialog.tsx`,
`App.tsx`, `styles.css`, `app/e2e_test.py`의 "키보드 내비게이션" 스텝):

- ~~1. KIND_COLORS가 CSS 토큰 밖~~ → `styles.css` `:root`에 `--kind-*` 토큰 추가,
  `KIND_COLORS`는 `var(--kind-*)` 참조만 보관(단일 소스 확보).
- ~~2. 그래프 키보드 내비게이션 부재~~ → `aria-activedescendant` 패턴(WAI-ARIA APG) 적용.
  `.song-pane`(`role="listbox"`)가 DOM 포커스를 유지한 채 방향키로 "논리적 활성 노드"
  (`.kbd-active`, `role="option"`)를 이동, 이미 있던 `selected` 상태와 통합되어 Enter 없이도
  상세패널이 따라감. `Shift/Alt/Ctrl+우클릭`(비교·체인복사)은 이번 패스 스코프 밖으로 남음
  (아래 "남은 갭" 참고).
- ~~4. 모달 위험도 구분 약함~~ → 파괴적 확인 모달(덮어쓰기/체인교체/트랙전송)에 `.modal-danger`
  (상단 danger 보더 + 제목 앞 ⚠) 적용, 정보성 모달(단축키/비교결과)은 중립 유지.
- ~~5. BulkApplyDialog 단일 긴 모달~~ → 번호 매긴 섹션(`.bulk-step`) + 상시 요약 바
  (`.bulk-summary-bar`, NN/g "visibility of system status")로 재구성. 위저드형 단계 게이팅은
  채택 안 함(필드 상호의존성 대비 이득 적어 YAGNI 판단).
- ~~6. 프리웜 텍스트 카운터뿐~~ → `.prewarm-bar` 채워지는 바 추가(텍스트 병기).
- ~~7. 그래프 뷰포트 리셋~~ → `SongPane.tsx`에 문서별 `Viewport` 저장(`viewportByDoc`)
  후 탭 전환 시 복원, 처음 보는 문서만 `fitView`.

검증: `npm run build`(tsc 클린) + 헤드리스 self-test(30/31 노드 유지) + E2E(신규 키보드 내비게이션
스텝 포함 전체 시나리오 PASS) + `pytest engine/tests`(583 passed, 엔진 무변경 확인).

### 남은 갭

- **비교/체인복사의 키보드 등가 기능 없음** — `Shift/Alt/Ctrl+우클릭`은 여전히 마우스 컨텍스트
  메뉴 이벤트 전용. 그래프 탐색·선택·복사(Ctrl+C)·붙여넣기·실행취소는 키보드로 가능해졌지만,
  이 세 기능은 별도 설계(키 조합 확정, 기존 우클릭 흐름과의 일관성)가 필요해 이번 스코프에서
  제외함.
- **상태바 텍스트 키워드 매칭의 취약성** — 새 상태 문구 추가 시 "실패/완료/복원됨/복사됨/열림"
  키워드 목록에 반영 안 하면 조용히 색이 안 붙는 회귀 가능(크래시는 없음). 이번 패스에서
  손대지 않음 — 여러 호출부(`setStatusMsg`)에 걸친 리팩터라 범위가 커서 별도 작업으로 분리.
- **그래프 확대/축소 외의 캔버스 자체 접근성** — React Flow 캔버스 자체(팬/줌 제스처, 미니맵)는
  여전히 마우스/휠 전용. 방향키 노드 이동은 뷰포트를 자동으로 따라가지 않는다(포커스가 화면
  밖으로 나가도 스크롤되지 않음) — 큰 그래프에서 계속 화살표로 이동하면 시각적으로 노드를
  놓칠 수 있음.
- **분할 레이아웃 미영속화** (2026-07-15 신설 기능의 알려진 한계) — 패널 배치(어떤 문서가 어느
  분할에 있는지, split 비율)는 세션 메모리에만 있고 앱 재시작 시 초기화됨. VSCode의 워크스페이스
  레이아웃 저장과 달리 이 앱은 QSettings 등에 레이아웃을 기록하지 않음 — 요청 스코프 밖(자유
  분할 자체가 목표였고, 영속화는 별도 요청 시 후속 작업).
- **분할 트리 키보드 조작 불가** — 패널 분할/이동/포커스 전환은 마우스(드래그) 또는 우클릭
  메뉴로만 가능. "다음 패널로 포커스 이동" 같은 키보드 단축키 없음.
